The big picture

A story ID enters; a finished MP4 leaves. Everything runs in one Celery worker, one stage at a time, one service call at a time — state lives in Redis, artifacts on disk. No step runs in parallel with another; this keeps a shared GPU cluster sane and the process easy to reason about.

The inference services

All models run on a home Kubernetes (k3s) cluster as cluster-internal services, reached from the local stack over tunnels. One LLM family — Nemotron — does triple duty (text, vision, speech-to-text).

Orchestration model

One worker, linear

A single Celery worker (concurrency=1) runs full_pipeline — which calls each stage in sequence, not as async sub-tasks. Within a stage, calls are sequential too (e.g. images render one section at a time). So the cluster only ever serves one request at a time.

Redis is the state

No SQL. Keys hold the run, the segment (script, plan, flags), per-section sections & images, source images, and a metrics record. Artifacts (audio, images, clips, the MP4) live on a shared outputs/ volume.

Why everything is non-fatal

A shared GPU can hiccup. Every optional service (source images, LTX, HyperFrames, music) is wrapped so a failure degrades gracefully — a missing motion clip just falls back to the image sequence — and the video still finishes.

the prompts

What each model is asked

Prompts are summarized. The pattern throughout: a tight system role + the story context, asking for one specific, structured thing.

① Summary & metadata — Nemotron LLM

# role Summarize this article for a tech-news short. Return a crisp summary, a 1-2 sentence hook, 2-6 tags, 4 emoji, and a haiku. # input {cleaned article text}

② Script — Nemotron LLM → generate_script_v1

# role Write a two-host podcast script about this story. Alternate speakers tagged [S1] and [S2]; natural, punchy, accurate. One idea per line; conversational; no stage directions. # input {summary} + {article}

③ Image prompt (per section) — Nemotron LLM → generate_image_prompt_v1

# role You are a cinematic art director writing ONE image prompt. Invent a striking, SPECIFIC scene: a clear subject doing something, a concrete setting, evocative props, a deliberate shot type. Describe ONLY scene + composition — do NOT mention art style. # then (deterministic) append the take's THEME style block so every shot in the video is stylistically cohesive.

④ The director's plan — Nemotron LLM → meta_sequencer

# role You are the director. For each narration section choose: image_sequence (default) · video (a hero motion moment) · hyperframe (structured kinetic text: keypoints/bigstat/quote/compare). # taste interesting, educational, funny, unexpected, original. most sections stay sequences; sprinkle ≤2 video & ≤2 hyperframe; never two specials back-to-back. # return JSON: one {template, recipe?, content?} per section.

A deterministic guardrail pass then enforces the caps/spacing, and a second LLM critic scores the plan on the taste rubric and revises weak spots.

⑤ Vision & ASR — Nemotron

# vision Describe this image in 1-2 sentences: subjects, mood. # asr Transcribe with word-level timestamps (for karaoke subtitles).

the agentic core

The director: how a section becomes a shot

Stage 4b is where hn.fm stops being a fixed template and starts making choices. It runs plan → guardrails → critic → guardrails, then produces only the special clips.

1 · Plan

LLM reads all sections + summary + theme + available source images, assigns a template to each.

→

2 · Guardrails

Deterministic: caps (≤2 each), no two specials adjacent, sequence is the floor. Fixes violations.

→

3 · Critic

A second LLM scores interest/originality, flags repetition, suggests swaps. Guardrails run again.

The three ways to tell a section

🖼️ image_sequence

The default. A root image, then FLUX image-to-image edits that evolve it — shown in quick cadence. Most sections.

🎞️ video

An LTX-2 motion clip from the section's still + a motion prompt, lightly stretched. A hero moment.

✨ hyperframe

HyperFrames kinetic text (keypoints / bigstat / quote / compare) over the image — for structured beats.

Every special clip is folded into the timeline as a video_clip_path with partial coverage: it plays its natural length, then the image sequence fills the rest of the section — so no downstream step needs to know which kind it was.

"These Are the Most Beautiful Equations"

Source: scientificamerican.com · theme Technical Blueprint · 16:9 · 21 sections. Tags mathematicsequationsgeometry  🧮 🎨 📐 🔢

① Summary (Nemotron)

Mathematicians view equations as artistic expressions of beauty — valuing simplicity, unexpected connections, and profound insights. The Loewner equation links complex analysis to statistical mechanics; Gauss-Bonnet ties curvature to topology…

② Script (two-host dialogue)

[S1] Welcome to Proof Positive, your science podcast. [S2] Today we're exploring the most beautiful equations mathematicians love. [S1] To them, equations are like art — simple, surprising, or deeply insightful. [S2] Let's start with the Loewner differential equation…

③ A real image prompt (§1, before the theme block)

A mathematician in a sunlit study stands before a massive chalkboard, meticulously drawing a single elegant equation that weaves together the Gauss-Bonnet formula and the Euler product, with a vintage compass… → + "Technical Blueprint" style appended deterministically

④ The director's plan (21 sections)

§1 SEQ · §2 HYPER · §3-9 SEQ · §10 VIDEO · §11-18 SEQ · §19 VIDEO · §20-21 SEQ

→ 18 sequences, 2 motion clips, 1 hyperframe. The hyperframe content the LLM wrote for §2:

keypoints · kicker "Math is art" · title "The Art of Equations" • Simple beauty in complex forms • Surprising connections across fields • Deep insights in minimal expressions

⑤ Real source images (Nemotron-described, from the article)

4 chalkboard photos pulled & described, e.g. "a complex mathematical formula for the partition function p(n) written in white chalk on a dark blackboard."

Where the time & tokens go

Measured per stage by the built-in metrics (visible live on the /observability dashboard). One ~3-minute, 19-section video:

Total render429s (~7 min) · ~13k tokens
scrape 9s source 0.2s script 10s audio 21s images 220s media_plan 79s assemble 90s
51%
images — the bottleneck. ~57 sequential FLUX calls (19 sections × root + edits).
21%
assemble — ffmpeg concat of ~30 timeline segments + subtitle burn + music mix.
18%
media_plan — mostly one LTX clip (~70s) + plan & critic LLM calls.

Levers

Fewer sequence frames (SEQ_MAX_FRAMES) cuts the 51% chunk most. Tokens concentrate in image-prompt generation. LTX reliability (a shared GPU) is the main reason a planned video sometimes drops to a sequence.
